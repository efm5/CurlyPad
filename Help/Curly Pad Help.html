The Menu

The menu has six items: “File”; “Edit”; “View”; “Options”; “Comment” and “Help”. The “Comment” menu may be hidden by the user. All six of these items have accelerator keys. All menu items have either accelerator keys (when they have sub–subitems) or keyboard shortcuts.

The File Menu (ALT+F)

New (CTRL+n)

“New” clears all text from the text box; if there is any text already in the text box, you will be offered the chance to save it.

New Window (CTRL+SHIFT+n)

“New Window” opens a new, empty Curly Pad window.

Openģ (CTRL+o)

“Openģ” opens a file requester allowing the user to choose a text file to open.

Recent (ALT+R)

“Recent” has a variable number of sub–subitems. Each of these items displays a fully qualified path to a recently edited text file. The maximum total number of items this list is adjustable by the user (1 minimum, 50 maximum). Files which no longer exist on disc you will be automatically removed from the list.

Save (CTRL+s)

“Save”

Save Asģ (CTRL+SHIFT+s)

“Save Asģ” opens a file requester allowing the user to name the text file to be saved and select its folder.

Page Set Upģ (CTRL+j)

“Page Set Upģ” opens the Windows built–in Page Setup dialog.

Printģ (CTRL+p)

“Printģ” opens the Windows built–in Print dialog.

Exit (ALT+F4)

“Exit” closes Curly Pad; if there is any text already in the text box, you will be offered the chance to save it.

The Edit Menu (ALT+E)

Undo (CTRL+z)

“Undo” undoes the last editing action. The undo stack is virtually unlimited (limited only by the amount of RAM in the computer).

Redo (CTRL+y)

“Redo” restores the last undone editing action. The redo stack is virtually unlimited (limited only by the amount of RAM in the computer).

Cut (CTRL+x)

“Cut” deletes the selection (moving it to the Clipboard); if nothing is selected nothing will be deleted.

Copy (CTRL+c)

“Copy” copies the selection (moving it to the Clipboard); if nothing is selected nothing will be copied.

Copy All (CTRL+SHIFT+c)

“Copy All” copies everything (moving it to the Clipboard); if nothing is selected everything will be copied anyway.

Copy To Beginning (CTRL+SHIFT+Home)

“Copy To Beginning” copies everything from the current cursor to the very beginning of the file (moving it to the Clipboard); selection is ignored.

Copy To End (CTRL+SHIFT+End)

“Copy To End” copies everything from the current cursor to the very end of the file (moving it to the Clipboard); selection is ignored.

Paste (CTRL+v)

“Paste” moves the Clipboard ’s textual contents into the text box at the current caret location. If there is no text content on the Clipboard nothing will be pasted.

Delete (Del the {DELETE} key)

“Delete” deletes the current selection or, if nothing is selected,the character immediately following the carrot location is deleted.

Trim To Beginning (CTRL+SHIFT+Del the {DELETE} key)

“Trim to Beginning” deletes everything from the current cursor to the very beginning of the file (NOT moving it to the Clipboard); selection is ignored.

Trim To End (CTRL+Alt+Del the {DELETE} key)

“Trim To End” deletes everything from the current cursor to the very end of the file (NOT moving it to the Clipboard); selection is ignored.

Search With Googleģ (CTRL+e)

“Search With Googleģ” open your preferred browser to the Google Advanced Search page. If something is selected in the text box, that selection will be copied to the Clipboard and thereafter pasted into the “all these words:” text field.

Findģ (CTRL+f)

“Findģ” opens the The Find Dialog.

Find Next (F3 the number three function key)

“Find Next” finds (forward) the next existing instance of the last string searched for with the Find dialog.

Find Previous (SHIFT+F3 the shifted number three function key)

“Find Previous” finds (backwards) the previous existing instance of the last string searched for with the Find dialog.

Both “Find Next” and “Find Next” wraparound. If the end of the file is reached while searching forward, the search will continue at the beginning of the file. If the beginning of the file is reached while searching backwards, the search will continue at the end of the file.

“Line Endings” opens the “Line Endings” submenu which has five sub–subitems: “\n\n to \r\n”; “\n to \r\n”; “\n\r to \r\n”; “\r\n to \n\n” and “double space”. Each of these sub–subitems converts the line endings to the specified format.

“\n to \r\n” (CTRL+ALT+Shift+3)

The View Menu (ALT+V)

Word Wrap (CTRL+w)

“Word Wrap” is a toggle. When checked, word wrapping is turned on and there will be no horizontal scrollbar. When not checked, word wrapping is turned off and there will be a horizontal scrollbar.

Always On Top (CTRL+SHIFT+t)

“Always On Top” is a toggle. When checked, Curly Pad will do its best to force itself to stay on top of all other windows. When not checked, Curly Pad will act in the traditional manner of ducking behind other active windows.

Windows is not a big fan of this “Always On Top” behavior! If two (or more) applications request this behavior the results will be unpredictable.

Fade In (CTRL+i)

“Fade In” is a toggle. When checked, Curly Pad’s will fade in over a very brief period of time at launch. This may help eyestrain and reduce flickering and flashing.

Zoom In (CTRL++ either the plus key on the number pad or keyboard)

“Zoom In” increases the size of the text box font by one point.

Zoom Out (CTRL+- either the minus/hyphen key on the number pad or keyboard)

“Zoom Out” decreases the size of the text box font by one point.

Zoom Default (CTRL+0)

“Zoom Default” restores the size of the text box spot to the last size specifically chosen by the user (not including the use of “Zoom In” nor “Zoom Out”). By default, this is 11 points.

Opacity (ALT+O opens the ōOpacityö subitem)

“Opacity” has four subitems: 30%; 50%; 75% and Opaque. Each controls the amount of opacity(transparency/translucency) of the main window. This might be useful if you are using Curly Pad to edit the text in an underlying application.

30% (ALT+3, CTRL+SHIFT+3) is extremely transparent (can easily be seen through).

50% (ALT+5, CTRL+SHIFT5) is moderately transparent.

Dark Theme (CTRL+d)

“Dark Theme” is a toggle. When its checked state changes (from on to off or off to on), it forces the entire color scheme to change. This completely overrides any custom color changes made by the user. The dark mode (checked) is white on black and the other mode is black on white.

If any custom colors have been chosen by the user, this item might need to be toggled two or three times to achieve the desired color mode.

The Coding Menu

Coding Preferences (CTRL+SHIFT+ALT+s)

“Coding Preferences” opens the “Coding Preferences” dialog; details may be found here.

Add Double (CTRL+SHIFT+ALT+d)

“Add Double” adds “//  ” (two forward slashes followed by two spaces) to the beginning of the selected text. If nothing is selected everything will be selected first. If “Include Whitespace” is checked in the “Comment Preferences” settings dialog, the forward slashes will be placed at the beginning of the line followed by any white space and then the text. If “Comment Out Blank Lines” is checked in “Comment Preferences”, blank lines will have forward slashes added to the line, otherwise they will remain blank.

If the text in the box looks like this (and nothing is selected, thus everything will be selected):

The quick brown fox
jumped over the lazy dog

It should look something like this after adding double:

//  The quick brown fox
//  jumped over the lazy dog


Add Triple (CTRL+SHIFT+ALT+t)

“Add Triple” adds “///  ” (three forward slashes followed by two or three* spaces) to the beginning of the selected text. If nothing is selected everything will be selected first. If “Include Whitespace” is checked in the “Comment Preferences” settings dialog, the forward slashes will be placed at the beginning of the line followed by any white space and then the text. If “Comment Out Blank Lines” is checked in “Comment Preferences”, blank lines will have forward slashes added to the line, otherwise they will remain blank.

* if “Use Three Spaces” is checked, three spaces will be used, otherwise two spaces will be used. This setting only applies to adding triple.

Make Summary (CTRL+SHIFT+ALT+m)

“Make Summary” adds a summary header to the selection and makes it a triple comment. If nothing is selected everything will be selected.

If the text in the box looks like this (and nothing is selected, thus everything will be selected):

The quick brown fox
jumped over the lazy dog

It should look something like this after adding a summary header:

///   ‹Summary›

///   The quick brown fox
///   jumped over the lazy dog


C++ (CTRL+SHIFT+ALT+c)

“C++” turns the selected text into an old–fashioned C– or C++–style comment. If nothing is selected everything will be selected.

If the text in the box looks like this (and nothing is selected, thus everything will be selected):

The quick brown fox
jumped over the lazy dog

It should look something like this after adding a summary header:

/*The quick brown fox
jumped over the lazy dog*/


Comment Out (CTRL+SHIFT+ALT+t)

“Comment Out” turns the selected text into a more modern C#–style comment. If nothing is selected everything will be selected. Note that there are no spaces after the slashes.

If the text in the box looks like this (and nothing is selected, thus everything will be selected):

The quick brown fox
jumped over the lazy dog

It should look something like this after adding a summary header:

//The quick brown fox
//jumped over the lazy dog


Remove (CTRL+SHIFT+ALT+r)

“Remove” removes all commenting marks.

Wrap (CTRL+SHIFT+ALT+w)

“Wrap” limits the number of words on a line. It uses two distinctly different algorithms. If “Concatenate First” is checked in the “Comment Preferences” dialog, the entire selection (if nothing is selected everything will be selected) will be concatenated into a single line; otherwise, all lines will be treated individually. Each line (there might only be one line)will be split at word boundaries so that no line exceeds the maximum allowed number of characters.

Extremely long words might cause problems if they are longer than the total number of characters allowed!

Add Not Implemented (CTRL+SHIFT+ALT+0 (zero))

“Add Not Implemented” modifies the current selection (if nothing is selected, everything will be selected). When Visual Studio automatically creates event handlers it creates function stubs which look something like this:

      private void XXX_Click(object sender, EventArgs e) {

      }

after performing “Add Not Implemented” the function stub would look something like this:

      private void XXX_Click(object sender, EventArgs e) {
         TimedMessage("XXX_Click", "NOT YET IMPLEMENTED");
      }

TimedMessage is a custom function which relies on P/Invoke:

      public static void TimedMessage(string pMessage, string pTitle = "", int pDuration = 4500) {
         _ = MessageBoxTimeout(IntPtr.Zero, pMessage, pTitle, TIMED_MESSAGEBOX_FLAGS, 0, pDuration);
      }

TimedMessage P/Invokes:

      [DllImport("user32.dll", CharSet = CharSet.Unicode, SetLastError = true)] public static extern int MessageBoxTimeout(IntPtr hWnd, String lpText, String lpCaption, uint uType, Int16 wLanguageId, Int32 dwMilliseconds);

TimedMessage uses these TIMED_MESSAGEBOX_FLAGS:

      private const uint TIMED_MESSAGEBOX_FLAGS = /*MB_OK*/ 0x00000000 | /*MB_SETFOREGROUND*/ 0x00010000 | /*MB_SYSTEMMODAL*/ 0x00001000 | /*MB_ICONEXCLAMATION*/ 0x00000030;

TimedMessage could be replaced with the standard MessageBox function but it is slightly more Dragon-friendly and includes the ability to set a timeout value.

Reverse Equality (CTRL+SHIFT+ALT+1 (one))

“Reverse Equality” modifies the current selection (if nothing is selected, everything will be selected). The sense of any equality statement is reversed. Something which looks like this:

      switch (sColorUsage) {
         case ColorUsage.AddressBarBackgroundColor:
            usageColor = sCurrentTheme.mAddressBarBackgroundColor;
            break;
         case ColorUsage.AddressTextBoxBackgroundColor:
            usageColor = sCurrentTheme.mAddressTextBoxBackgroundColor;
            break;
      }

after performing “Reverse Equality” the selection would look like this:

      switch (sColorUsage) {
         case ColorUsage.AddressBarBackgroundColor:
            sCurrentTheme.mAddressBarBackgroundColor = usageColor;
            break;
         case ColorUsage.AddressTextBoxBackgroundColor:
            sCurrentTheme.mAddressTextBoxBackgroundColor = usageColor;
            break;
      }

The lines which look like:
usageColor = sCurrentTheme.mAddressBarBackgroundColor;
have their equality reversed:
sCurrentTheme.mAddressBarBackgroundColor = usageColor;

Expression-Bodied Method (CTRL+SHIFT+ALT+8)

“Expression-Bodied Method” modifies the current selection (if nothing is selected, everything will be selected).

      private void XXX() {
            DoSomething();
      }

after performing “Expression-Bodied Method” the function would look something like this:

      private void XXX() => DoSomething();

The Status Bar

The Status Bar displays the number of characters, words, and lines; it displays the text encoding (e.g. UTF–8). The four–digit Curly Pad version number is the last status item.

The Dialogs

The Find Dialog

The “Find” dialog has a single text entry field – enter the phrase for which you wish to search in this text box. To the right of the text box is the “Find Next” button; when clicked, using the criteria specified in the “Options” and “Scope” GroupBoxes, the next instance of this phrase will be brought into view and highlighted. This “Find” dialog will close automatically if the phrase is found; if it is not found the dialog will remain open. Clicking the Cancel button will close this dialog without searching for anything.

The “Options” GroupBox presents the user with three CheckBoxes: “Match Case”; “Regular Expressions” and “Whole Word”.

“Match Case”, when checked, forces the search to be case-sensitive (e.g. when checked, “something” will not match with “Something”; when not checked they will match).

“Regular Expressions”, when checked, allows the use of Regular Expressions when searching.

The Microsoft Regular Expressions tutorial may be found here.

“Whole Word”, when checked, forces the search to be limited to whole words (e.g. when checked, if you search for “some” it will not find “something”).

Currently, “Whole Words” is only implemented for “Regular Expressions”. Note that it is displayed underneath “Regular Expressions” and slightly indented. When this feature is implemented for all searches it will be moved above “Regular Expressions” and not indented.

The “Scope” GroupBox presents the user with two RadioButtons: “Selection” and “Global”. These items are mutually exclusive – only one may be selected.

“Selection”, when selected, forces the search to be restricted to the selection – if nothing is selected, nothing will be found.

“Global”, when selected, forces the entire text to be searched.

The Search And Replace Dialog

The “Search And Replace” dialog has two text entry field – enter the phrase you wish to replace in the “Search For:” text box; enter the replacement phrase in the “Replace With:” text box. To the right of these text boxes are the “Replace” and “Replace All” buttons. Wwhen clicked, the “Replace” button (using the criteria specified in the “Options” and “Scope” GroupBoxes) replaces the next instance of the phrase with the replacement phrase. The “Replace All” button replaces all instances of the phrase with the replacement phrase. This “Search And Replace” dialog closes automatically if any phrase is replaced; if nothing is replaced the dialog will remain open. Clicking the Cancel button will close this dialog without replacing anything.

The “Options” and “Scope” GroupBoxes preform exactly like those in the Find dialog; details may be found here (Options GroupBox) and here (Scope GroupBox).

The Go To Line Dialog

The “Go To” dialog allows the user to bring a specific line into view. This is only useful if the user knows the line number for the line of interest.

Experimental code has been removed which added rudimentary line numbering functionality. Getting this working completely and correctly has proved very challenging! The feature may return in some future version.

The Font Size Dialog

The Font Size dialog allows the user to adjust the size of either the interface (GUI) or text box font. This may be done dynamically or statically. When the “Close On Okay” is checked, the behavior will be static (the dialog will close when the “OK” button is clicked). When it is not checked, the behavior is dynamic – the font size change is applied but the dialog remains open so that additional sizes may be tried.

The minimum font size is eight (8); the maximum font size is five hundred twelve (512).

Larger font sizes may be achieved for the text box using the “Zoom In” feature or controlled mouse wheel scrolling.

The Maximum File History Dialog

The Maximum File History dialog allows the user to control the maximum number of recently opened files which are displayed and stored. The minimum value is zero (0 – effectively turning the feature off), the maximum value is fifty (50).

The Coding Preferences Dialog

The “Coding Preferences” dialog is only of interest to programmers. It presents a number of settings which control how comments are managed.

The “Comment Width” GroupBox has a numeric entry widget which controls the number of characters–per–line of a comment which has been automatically wordwrapped. The “Concatenate First” controls whether comments are created on a line–by–line basis or if the entire selection is concatenated into a single before wordwrapping.

The “Use Three Spaces” CheckBox controls the number of spaces after three slashes. When checked, three spaces will be added after a three slash comment; otherwise, two spaces will be added.

The “Include Whitespace” CheckBox controls whether not the commenting slashes are placed at the extreme front of the line. When checked, the commenting slashes are placed at the very beginning of the line; when not checked, the commenting slashes are placed immediately in front of the first non–whitespace of the line.

The “Comment Out Blank Lines” CheckBox controls whether or not blank lines get commenting slashes. When checked, blank lines are commented out; otherwise, blank lines did not get any commenting slashes.

The “Tabs Or Spaces” GroupBox contains two RadioButtons: “Use Tabs” and “Use Spaces”; they are mutually exclusive. When “Use Tab” is checked, all sets of spaces (as specified in the numeric widget below) will be turned into tabs. When it is not checked, all actual tabs will be turned into spaces (again as specified in the numeric widget). The numeric widget As a minimum value of two (2) and a maximum value of ten (10).

The Create Theme Dialog

The Create Theme dialog allows the user to give a name to the currently existing theme options. Theme names must be unique, they are not case-sensitive. Themes are stored in a plain text file located in the users “Documents” folder, in a subfolder called: “Curly_Pad_Data”. There is no limit to the number of themes which may be created.

The Edit Theme Dialogs

The Edit Theme picker dialog presents all of the existing themes (except “Dark” and “Light” – which are built-in and may not be edited). Choosing any of these opens the actual theme editor dialog.


The Theme Editor dialog presents the selected theme for editing. All of the theme’s features may be changed. Each color swatch reflects the color associated with the button immediately to its left. These color swatches may be clicked just like the button to which it is associated.

This dialog’s layout design may change slightly in the future. Status Bar background color and dialog background color items might move into the Interface GroupBox.

The Remove Theme Dialog

The Remove Theme dialog presents all of the existing themes (except “Dark” and “Light” – which are built-in and may not be removed). Choosing any of these immediately removes the theme – this is a destructive action and may not be undone.

Curly Pad’s Cleaner

This “Cleaner” will remove Curly Pad’s cache folder(s) and its recent file history. You may preserve your recent files history by renaming its text file which is located in the folder in which Curly Pad is installed. The file name is: “RecentFileHistory.txt”–just rename it to something obvious like “XRecentFileHistory.txt” before you run the “Cleaner”; then, after the cleaner has finished, rename it back to its original name: “RecentFileHistory.txt”.
